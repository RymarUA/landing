# Инструкция для диагностики проблемы с Новой Почтой

## Шаг 1: Откройте консоль браузера
1. Нажмите F12 или Ctrl+Shift+I
2. Перейдите на вкладку **Console**
3. Очистите консоль (иконка 🚫 или Ctrl+L)

## Шаг 2: Оформите тестовый заказ
1. Добавьте товар в корзину
2. Перейдите в корзину
3. Нажмите "Оформити замовлення"
4. Заполните форму:
   - Имя: Test
   - Телефон: +380501234567
   - Email: test@test.com
   - Выберите город (например, Київ)
   - Выберите отделение Новой Почты
5. Нажмите "Оформити замовлення"

## Шаг 3: Скопируйте ВСЕ логи из консоли
Ищите логи с префиксами:
- `[checkout]` - логи с формы оформления
- `[/api/checkout]` - логи с сервера
- `[sitniks]` - логи интеграции с CRM

## Шаг 4: Проверьте Network
1. Перейдите на вкладку **Network** в DevTools
2. Найдите запрос к `/api/checkout`
3. Посмотрите:
   - Request Payload (что отправляется)
   - Response (что возвращается)

## Что нужно скопировать:
```
1. Все логи из Console (особенно с [checkout] и [sitniks])
2. Request Payload из Network → /api/checkout → Payload
3. Response из Network → /api/checkout → Response
```

## Важно!
Логи при загрузке страницы (`First page returned no products`) - это нормально, если в CRM нет товаров.
Нам нужны логи **во время создания заказа**.
