# Автоматичне створення ТТН Нової Пошти

## 🎯 Що це дає?

Після оформлення замовлення на сайті **автоматично створюється ТТН (експрес-накладна)** в Новій Пошті.

- ✅ Замовлення створюється в Sitniks CRM
- ✅ ТТН створюється через API Нової Пошти
- ✅ Номер ТТН повертається клієнту
- ✅ Менеджеру не потрібно вручну створювати ТТН

---

## 📋 Налаштування

### Крок 1: Отримайте дані відправника з Нової Пошти

Зайдіть в особистий кабінет Нової Пошти: https://my.novaposhta.ua

#### 1.1 Місто відправника (CityRef)
1. Перейдіть в **Контрагенти** → **Відправники**
2. Виберіть вашу компанію
3. Скопіюйте **Ref міста** (наприклад: `8d5a980d-391c-11dd-90d9-001a92567626` для Києва)

#### 1.2 Відділення відправника (WarehouseRef)
1. В розділі **Відправники** знайдіть ваше відділення
2. Скопіюйте **Ref відділення**

#### 1.3 Контактна особа (ContactRef)
1. В розділі **Контрагенти** → **Контактні особи**
2. Знайдіть вашу контактну особу
3. Скопіюйте **Ref контактної особи**

#### 1.4 Телефон відправника
Ваш номер телефону в форматі `+380XXXXXXXXX`

---

### Крок 2: Заповніть .env.local

Відкрийте файл `.env.local` та заповніть:

```env
# Налаштування відправника для автоматичного створення ТТН
NOVAPOSHTA_SENDER_CITY_REF=8d5a980d-391c-11dd-90d9-001a92567626  # Ваш Ref міста
NOVAPOSHTA_SENDER_WAREHOUSE_REF=ваш-ref-відділення                # Ваш Ref відділення
NOVAPOSHTA_SENDER_CONTACT_REF=ваш-ref-контакту                    # Ваш Ref контакту
NOVAPOSHTA_SENDER_PHONE=+380501234567                              # Ваш телефон
```

---

### Крок 3: Перезапустіть сервер

```bash
npm run dev
```

---

## 🧪 Тестування

### 1. Оформіть тестове замовлення:
- Додайте товар в корзину
- Перейдіть до оформлення
- Заповніть форму (обов'язково виберіть місто та відділення НП)
- Оформіть замовлення

### 2. Перевірте логи в терміналі:
```
[/api/checkout] Створення ТТН для замовлення 12345
[novaposhta-ttn] Створення ТТН: {...}
[novaposhta-ttn] ✓ ТТН створено: 20450123456789
[/api/checkout] ✓ ТТН створено: 20450123456789
```

### 3. Перевірте в Новій Пошті:
- Зайдіть в особистий кабінет НП
- Перейдіть в **Експрес-накладні**
- Знайдіть створену ТТН за номером

## 🔍 Діагностика проблем

### Помилка: "Налаштування відправника не заповнені"

**Причина:** Не заповнені змінні в `.env.local`

**Рішення:**
1. Перевірте, що всі 4 змінні заповнені:
   - `NOVAPOSHTA_SENDER_CITY_REF`
   - `NOVAPOSHTA_SENDER_WAREHOUSE_REF`
   - `NOVAPOSHTA_SENDER_CONTACT_REF`
   - `NOVAPOSHTA_SENDER_PHONE`
2. Перезапустіть сервер

---

### Помилка від API Нової Пошти

**Можливі причини:**
- Невірний Ref міста/відділення/контакту
- Невірний формат телефону
- Недостатньо коштів на рахунку НП
- Проблеми з API ключем

**Рішення:**
1. Перевірте логи в терміналі - там буде детальна помилка від НП
2. Перевірте правильність Ref в особистому кабінеті НП
3. Перевірте баланс рахунку в НП

---

### ТТН не створюється, але помилок немає

**Причина:** Клієнт не вибрав місто або відділення НП

**Рішення:**
- Переконайтесь, що на формі оформлення вибрано:
  - Місто доставки
  - Відділення Нової Пошти

---

## 📊 Як це працює?

### Для замовлень з накладеним платежем (COD):
1. Клієнт оформляє замовлення
2. Створюється замовлення в Sitniks CRM
3. **Автоматично створюється ТТН** через API НП
4. Номер ТТН повертається клієнту
5. ТТН з'являється в особистому кабінеті НП

### Для онлайн-оплати:
1. Клієнт оформляє замовлення
2. Перенаправлення на WayForPay
3. Після оплати → створюється замовлення в Sitniks
4. **Автоматично створюється ТТН** (webhook від WayForPay)
