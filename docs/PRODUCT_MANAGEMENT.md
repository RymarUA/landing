# Управління товарами в Sitniks CRM

## Огляд

Система створення товарів дозволяє адміністраторам додавати товари в Sitniks CRM з підтримкою варіацій (розміри, кольори) та повною інтеграцією з існуючою інфраструктурою.

## Доступ до форми

1. Перейдіть в адмін-панель: `/admin`
2. Натисніть "📦 Управління товарами" або перейдіть за прямим посиланням: `/admin/products`
3. Заповніть форму створення товару

## Структура форми

### 1. Базова інформація

- **Назва товару** - Обов'язкове поле, відображається в каталозі
- **Категорія** - Обов'язкове поле для групування товарів
- **Опис** - Детальний опис товару для клієнтів

### 2. Ціни та ідентифікатори

- **SKU** - Унікальний артикул товару (обов'язково)
- **Штрихкод** - Опціонально, для сканування
- **Ціна продажу** - Роздрібна ціна в грн (обов'язково)
- **Ціна закупівлі** - Оптова ціна для розрахунку маржі

### 3. Фізичні характеристики

- **Вага (кг)** - Вага для розрахунку доставки Новою Поштою
- **Розміри (см)** - Ширина, довжина, висота для упаковки

### 4. Варіації товару

Коли увімкнено варіації, можна створювати різні версії товару:

#### Розміри
- **Швидкі розміри:** XS, S, M, L, XL, XXL, 3XL
- **Кастомні розміри:** Введіть будь-який розмір (наприклад, 42, 44, 46)

#### Кольори
- **Популярні кольори:** Чорний, Білий, Сірий, Червоний, Синій, Зелений, Жовтий, Рожевий
- **Кастомні кольори:** Введіть будь-який колір

#### Унікальні SKU для варіацій
- Кожна варіація повинна мати унікальний SKU
- Автоматично генерується на основі базового SKU
- Можна редагувати вручну

#### Ціни та кількість
- **Ціна** - Може відрізнятися від базової
- **Кількість** - Кількість на складі
- **Штрихкод** - Унікальний для кожної варіації

### 5. Зображення

- Завантаження зображень товару
- Підтримка PNG, JPG до 10MB
- Drag & drop інтерфейс

## Як працюють варіації

### Приклад 1: Футболка з розмірами

```
Базовий товар: Футболка класична
SKU: TSHIRT-001

Варіації:
- TSHIRT-001-S (Розмір S, Ціна 500грн)
- TSHIRT-001-M (Розмір M, Ціна 550грн) 
- TSHIRT-001-L (Розмір L, Ціна 600грн)
- TSHIRT-001-XL (Розмір XL, Ціна 650грн)
```

### Приклад 2: Телефон з кольорами

```
Базовий товар: Смартфон Pro
SKU: PHONE-001

Варіації:
- PHONE-001-BLACK (Колір Чорний, Ціна 12000грн)
- PHONE-001-WHITE (Колір Білий, Ціна 12000грн)
- PHONE-001-BLUE (Колір Синій, Ціна 12500грн)
```

### Приклад 3: Складні варіації (розмір + колір)

```
Базовий товар: Кросівки Sport
SKU: SHOES-001

Варіації:
- SHOES-001-S-BLACK (Розмір S, Колір Чорний, Ціна 2500грн)
- SHOES-001-M-BLACK (Розмір M, Колір Чорний, Ціна 2600грн)
- SHOES-001-L-BLACK (Розмір L, Колір Чорний, Ціна 2700грн)
- SHOES-001-S-WHITE (Розмір S, Колір Білий, Ціна 2500грн)
```

## Інтеграція з Sitniks CRM

### Автоматична синхронізація

1. **Створення товару** - Товар з'являється в Sitniks CRM
2. **Варіації** - Створюються як окремі варіації товару
3. **Склад** - Кількість автоматично синхронізується
4. **Ціни** - Роздрібна та закупівельна ціни

### Поля в Sitniks

```
Основний товар:
- title: Назва товару
- description: Опис
- price: Базова ціна
- costPrice: Ціна закупівлі
- sku: Базовий SKU
- barcode: Штрихкод
- weight: Вага
- isActive: Активність
- category: Категорія

Варіації:
- sku: Унікальний SKU
- barcode: Штрихкод варіації
- price: Ціна варіації
- costPrice: Ціна закупівлі
- weight: Вага варіації
- properties: Характеристики (Розмір, Колір)
- warehouseQuantities: Кількість на складі
```

## Валідація

### Обов'язкові поля

- Назва товару
- Категорія
- SKU (для товару та кожної варіації)
- Ціна (> 0)

### Правила валідації

- Ціна закупівлі не може бути від'ємною
- Вага не може бути від'ємною
- Кожна варіація повинна мати розмір або колір
- SKU варіацій повинні бути унікальними

## Поширені помилки

### 1. SKU вже існує
```
Помилка: "Product with this SKU already exists"
Рішення: Змініть SKU на унікальний
```

### 2. Від'ємні значення
```
Помилка: "Price cannot be negative"
Рішення: Перевірте ціни та вагу
```

### 3. Пусті варіації
```
Помилка: "Each variation must have size or color"
Рішення: Додайте розмір або колір до кожної варіації
```

## Кращі практики

### 1. Система найменування SKU

```
Формат: {CATEGORY}-{MODEL}-{VARIATION}
Приклади:
- TSHIRT-001-S (Футболка, модель 001, розмір S)
- PHONE-PRO-BLACK (Телефон, Pro, колір чорний)
- SHOES-SPORT-42-BLUE (Кросівки, Sport, розмір 42, колір синій)
```

### 2. Стратегія цін

```
Базова ціна: 500грн
Розмір M: +10% (550грн)
Розмір L: +20% (600грн)
Розмір XL: +30% (650грн)
```

### 3. Управління складом

- Встановлюйте реалістичну кількість
- Використовуйте одиниці для кожної варіації
- Регулярно оновлюйте кількість

## Інтеграція з іншими функціями

### 1. Корзина покупок
- Варіації відображаються як опції вибору
- Кожна варіація має свою ціну та кількість

### 2. Оформлення замовлення
- Вага варіації використовується для розрахунку доставки
- SKU варіації передається в Sitniks

### 3. Аналітика
- Статистика по кожній варіації
- Популярні розміри та кольори
- Маржинальність по варіаціях

## API

### Створення товару

```typescript
POST /open-api/products
{
  "title": "Футболка класична",
  "description": "Якісна футболка з бавовни",
  "price": 500,
  "costPrice": 200,
  "weight": 0.5,
  "sku": "TSHIRT-001",
  "isActive": true,
  "variations": [
    {
      "sku": "TSHIRT-001-S",
      "price": 500,
      "costPrice": 200,
      "weight": 0.5,
      "properties": [
        { "name": "Розмір", "value": "S" }
      ]
    }
  ]
}
```

### Отримання товарів

```typescript
GET /open-api/products?limit=50&skip=0
```

### Оновлення товару

```typescript
PATCH /open-api/products/{id}
{
  "price": 550
}
```
