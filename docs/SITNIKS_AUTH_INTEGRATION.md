# Інтеграція авторизації з Sitniks CRM

Тепер авторизація на сайті автоматично синхронізується з Sitniks CRM. Кожен клієнт створюється/оновлюється в CRM при вході.

## Як це працює

### При реєстрації/вході:
1. **Email OTP** → клієнт вводить email і отримує код
2. **Автоматичний пошук** → система шукає клієнта в Sitniks по email
3. **Створення/оновлення** → якщо не знайдено — створює нового, якщо знайдено — оновлює дані
4. **JWT токен** → зберігає ID клієнта з Sitniks в токені
5. **Синхронізація** → всі дані автоматично синхронізуються

### В особистому кабінеті:
- **Статистика покупок** з Sitniks (кількість замовлень, витрачено, середній чек)
- **Історія замовлень** в реальному часі
- **Дані клієнта** (дата реєстрації, контактна інформація)

## Налаштування

### 1. Переменні середовища

Додайте в `.env.local`:

```bash
# Sitniks CRM інтеграція
SITNIKS_API_URL=https://your-store.sitniks.com/api/v1
SITNIKS_API_KEY=your_bearer_token_here

# JWT для сесій
JWT_SECRET=your_secret_key_here

# Email сервіс (Resend)
RESEND_API_KEY=your_resend_api_key
EMAIL_FROM=FamilyHub Market <noreply@yourdomain.com>
```

### 2. Sitniks CRM налаштування

1. **API доступ**:
   - Увійдіть в Sitniks
   - Налаштування → API
   - Отримайте Bearer token
   - Додайте IP адресу сервера в білий список (якщо потрібно)

2. **Поля клієнта**:
   - Переконайтеся що поля `email`, `phone`, `fullname` активні
   - Налаштуйте статуси замовлень
   - Включіть ID інтеграції Нової Пошти

## Процес синхронізації

### Автоматична при вході:
```
Email → OTP → Перевірка → Пошук в Sitniks → Створення/оновлення → JWT з ID
```

### Оновлення телефону:
```
Профіль → Додати телефон → Оновлення в Sitniks → JWT оновлено
```

### Замовлення:
```
Чекаут → Створення замовлення → Sitniks клієнт → Автоматично прив'язується
```

## API ендпоінти

### `/api/auth/verify-otp`
- **POST** — перевірка OTP коду
- Автоматично синхронізує з Sitniks
- Повертає JWT з `sitniksCustomerId`

### `/api/profile/customer`
- **GET** — отримання даних клієнта з Sitniks
- Включає статистику покупок
- Тільки для автентифікованих користувачів

### `/api/auth/update-phone`
- **POST** — оновлення телефону клієнта
- Оновлює дані в Sitniks
- Оновлює JWT токен

## Дані що синхронізуються

### Клієнт:
- **Email** — основний ідентифікатор
- **Телефон** — для замовлень та доставки
- **Ім'я** — з профілю або з email
- **Коментар** — "Зареєстрований через сайт"

### Статистика:
- **Кількість замовлень** — всі замовлення клієнта
- **Загальна сума** — тільки виконані замовлення
- **Середній чек** — середня вартість замовлення
- **Останнє замовлення** — дата останнього замовлення
- **Дата реєстрації** — коли клієнт створений в CRM

## Обробка помилок

### Sitniks недоступний:
- Логін працює без синхронізації
- Клієнт може увійти в профіль
- Повторна спроба синхронізації при наступному вході

### Помилка API:
- Логування помилок в консоль
- Не блокує процес входу
- Graceful degradation

## Переваги інтеграції

### Для бізнесу:
- **Єдина база клієнтів** — сайт і CRM синхронізовані
- **Автоматична обробка** — без ручного введення даних
- **Аналітика** — повна статистика в реальному часі
- **Сегментація** — можливість працювати з клієнтами в CRM

### Для клієнта:
- **Безшовний досвід** — реєстрація за 30 секунд
- **Особистий кабінет** — з повною історією покупок
- **Актуальні дані** — завжди свіжі замовлення та статуси

## Моніторинг

### Логи:

- Кількість нових клієнтів
- Кількість оновлень даних
- Помилки API Sitniks
