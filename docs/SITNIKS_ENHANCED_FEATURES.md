# Sitniks CRM Enhanced Features

## Огляд

Цей документ описує розширені можливості інтеграції з Sitniks CRM, які дозволяють передавати більше даних про замовлення та клієнтів для кращої аналітики та управління.

## Нові поля в замовленнях

### 1. Email клієнта

**Поле:** `client.email`

Тепер форма оформлення замовлення збирає email клієнта (опціонально). Це дозволяє:
- Відправляти email-підтвердження замовлень
- Вести email-маркетинг
- Покращити ідентифікацію клієнтів

**Де налаштувати в Sitniks:**
- Налаштування → Клієнти → Поля клієнта

### 2. Коментар клієнта

**Поле:** `clientComment`

Клієнт може залишити коментар до замовлення (наприклад, "Передзвоніть перед відправкою", "Зручний час доставки 14:00-18:00").

**Відображення:** Поле "Коментар клієнта" в картці замовлення

### 3. Коментар менеджера (розширений)

**Поле:** `managerComment`

Автоматично генерується з наступною інформацією:
- Спосіб оплати (картка/накладений платіж)
- UserID (якщо клієнт авторизований)
- UTM-мітки (source, medium, campaign)
- Referrer (звідки прийшов клієнт)
- Знижка (якщо застосовано промокод)

**Приклад:**
```
Замовлення з сайту. Оплата: картка. UserID: 12345. UTM: source=google, medium=cpc, campaign=spring_sale. Referrer: https://google.com. Знижка: 150 грн
```

### 4. Вага відправлення (автоматична)

**Поле:** `npDelivery.weight`

Система автоматично розраховує вагу на основі товарів у замовленні:
- Використовує поле `weight` з карточки товару в Sitniks
- Якщо вага не вказана, використовує 0.5 кг за замовчуванням
- Мінімальна вага: 0.5 кг (вимога Нової Пошти)

**Формула:** `totalWeight = sum(productWeight * quantity)`

### 5. Опис відправлення (покращений)

**Поле:** `npDelivery.description`

Тепер включає кількість кожного товару:
```
Комплект 401 Valhalla x2, Плед x1, Подушка 50x70 x2
```

### 6. External ID

**Поле:** `externalId`

Унікальний ідентифікатор замовлення для уникнення дублікатів:
```
web-1710766800000-user-12345
```

Формат: `web-{timestamp}-user-{userId}`

## Налаштування Custom Fields для клієнтів

Для збереження аналітики та статистики клієнтів використовуються кастомні поля:

### Статистичні поля

1. **orders_count** - Кількість замовлень
2. **total_spent** - Загальна сума покупок (грн)
3. **last_order_at** - Дата останнього замовлення

### Поля джерела трафіку (для першого замовлення)

4. **first_utm_source** - UTM Source першого замовлення
5. **first_utm_medium** - UTM Medium першого замовлення
6. **first_utm_campaign** - UTM Campaign першого замовлення
7. **first_referrer** - Referrer першого замовлення

### Як налаштувати в Sitniks:

1. Перейдіть в **Налаштування → Кастомні поля → Клієнти**
2. Створіть нові поля з кодами вказаними вище
3. Тип поля: `text` для всіх полів
4. Модель: `clients`

## Налаштування статусів замовлень

Система автоматично встановлює різні статуси залежно від способу оплати:

### Змінні оточення:

```bash
# ID статусу "Новий" (для накладеного платежу)
SITNIKS_NEW_STATUS_ID=1

# ID статусу "Очікує оплати" (для онлайн-оплати)
SITNIKS_PENDING_PAYMENT_STATUS_ID=2

# ID статусу "Оплачено" (встановлюється після успішної оплати)
SITNIKS_PAID_STATUS_ID=3

# ID каналу продажів "Сайт"
SITNIKS_SALES_CHANNEL_ID=1

# ID інтеграції Нової Пошти
SITNIKS_NP_INTEGRATION_ID=1
```

### Як знайти ID в Sitniks:

1. **Статуси:** Налаштування → Статуси замовлень → ID в URL або таблиці
2. **Канали продажів:** Налаштування → Канали продажів → ID в URL
3. **Інтеграція НП:** Налаштування → Інтеграції → Нова Пошта → ID

## UTM-трекінг

Система автоматично збирає UTM-параметри з URL та зберігає їх:

### Підтримувані параметри:

- `utm_source` - Джерело трафіку (google, facebook, instagram)
- `utm_medium` - Канал (cpc, social, email)
- `utm_campaign` - Назва кампанії (spring_sale, black_friday)

### Приклад URL:

```
https://yoursite.com/?utm_source=google&utm_medium=cpc&utm_campaign=spring_sale
```

### Де зберігається:

1. **Коментар менеджера** - для всіх замовлень
2. **Custom Fields клієнта** - тільки для першого замовлення (для аналізу джерел залучення)

## Troubleshooting

### Вага не розраховується

**Проблема:** Вага завжди 0.5 кг

**Рішення:** Перевірте, що у товарів в Sitniks заповнено поле `weight` (вага в кг)

### UTM не зберігаються

**Проблема:** UTM-параметри не відображаються в коментарі

**Рішення:** Переконайтеся, що URL містить параметри `utm_source`, `utm_medium`, `utm_campaign`

### Custom Fields не створюються

**Проблема:** Статистика клієнта не оновлюється

**Рішення:** 
1. Створіть кастомні поля в Sitniks вручну
2. Перевірте права доступу API-ключа
3. Перевірте логи сервера на помилки

### Email не зберігається

**Проблема:** Email клієнта порожній в Sitniks

**Рішення:** Перевірте, що клієнт заповнив поле email в формі (воно опціональне)
